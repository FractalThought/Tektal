# Djupdykning i javaScript

## Ternary (Conditional Operator)

[Läs mer om ternary på MDN Web Docs](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Conditional_Operator)

En så kallad ternary är ett snabbt sätt att skapa en enkel kontrollsats, istället för att använda en if-else-sats.

För att skapa en sådan så använder vi följande syntax:

```js
condition ? doIfTrue : doIfNotTrue
```

Exempel:

```js
let age = 19
let myndig = age >= 18 ? "Ja" : "Nej"
```

Jämförelse med if-else:

```js
// if-else
if (condition) {
  //value if true;
} else {
  //value if false;
}
// Ternary
condition ? /* value if true */ : /* value if false */
```

Ternary använder vi framförallt när vi behöver göra små if-else-satser, oftast för att sätta ett värde. Använd det inte för att styra stora delar av din kod - då är det bättre att använda

En ternary fungerar på samma sätt som den här funktionen:

```js
ternaryFunction(age >= 18, "Ja", "Nej")

function ternaryFunction(condition, isTrue, isFalse) {
  if (condition) {
    return isTrue
  } else {
    return isFalse
  }
}
```

## Short-circuit evaluation

Läs mer om [short circuit evaluation på MDN Web Docs](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Logical_Operators#Short-circuit_evaluation).

De logiska operatorerna && och || använder vi vanligtvis i kontrollsatser när vi vill lägga till flera villkor, där && står för AND, och || står för OR.

AND innebär att det blir sant om båda villkoren på vardera sida om &&-operatorn är sant.

OR innebär att det blir sant om något av villkoren på vardera sida om ||-operatorn är sant.

Vi kan använda dessa operatorer för så kallade short-circuit evaluations, alltså en enkel if-sats. Skillnaden mellan dessa och en ternary är att vi inte behöver en else-sats.

Syntaxen för en sådan är följande:

```js
condition && /* körs när condition == true */

condition || /* körs när condition == false */
```
